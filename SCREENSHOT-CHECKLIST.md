# NitpickAI Showcase Screenshot Checklist

Use this checklist to capture portfolio-ready screenshots for GitHub, Upwork, and LinkedIn.

## Capture standards

- Use desktop resolution: **1440x900** or **1920x1080**
- Keep browser zoom at **100%**
- Use same visual theme in every shot
- Keep the header/logo fully visible in hero shots
- Prefer realistic text samples (not lorem ipsum)
- Blur or crop anything sensitive

## Required screenshots (minimum 6)

### 1) Hero / app overview
- Full app frame with branded header visible
- Show clean initial state before analysis
- Filename suggestion: `01-hero-editor.png`

### 2) Input with intentional errors
- Paragraph pasted with realistic grammar/spelling mistakes
- Show enough text to look credible
- Filename suggestion: `02-input-errors.png`

### 3) Analysis running state
- Show progress/loading state so clients see process, not just result
- Filename suggestion: `03-analysis-progress.png`

### 4) Highlighted issues + issue detail
- Show colored highlights and side panel/card detail together
- This is the money shot for product understanding
- Filename suggestion: `04-highlighted-issues.png`

### 5) Accept/ignore interaction
- Show one accepted and one ignored suggestion flow
- Proves user remains in control
- Filename suggestion: `05-accept-ignore.png`

### 6) Final cleaned output
- Show final text after review
- Keep message: cleaner writing, same voice
- Filename suggestion: `06-final-output.png`

## Optional screenshots (for technical buyers)

### 7) Admin/testing dashboard
- User list, tester status, or issue intake screen
- Filename suggestion: `07-admin-view.png`

### 8) Mobile/responsive snapshot
- Optional credibility shot for frontend quality
- Filename suggestion: `08-mobile-view.png`

### 9) API/docs or architecture visual
- If included, keep it clean and high-level
- Filename suggestion: `09-architecture.png`

## Suggested README placement order

1. Hero
2. Highlighted issues
3. Accept/ignore
4. Final output
5. (Optional) Admin

## Asset folder structure


## Notes for your Loom recording day

- Use the same sample paragraph from screenshots
- Keep recording under 90 seconds for primary version
- Show one clear before/after moment
- End with a direct CTA for client messages
